Topics Primarily Covered:

1. User Management
	1.1 Why is user management needed in Linux?
		Linux is a multi-user system, every process or task that runs in the operating system is run under a user. This user may be a human or a system user. Therefore, it is important to keep track of the users. Moreover, file ownership is dependant on permissions which are assigned to users, this maintains privacy and security.
	1.2 Where are user specific files and configurations stored? Give examples of the type of files that may be stored there.
		User specific files and configurations are stored in the /home/username directory. Common exmaples of the types of files and configurations kept here are:
			1.2.1 Shell configuration.
			1.2.2 SSH
			1.2.3 App configuration (git config, vim editor settings).
			1.2.4 Desktop environments (your extensions and themes in ubuntu).
	1.3 What does the Linux OS use for permission related tasks instead of user names?
		To configure permission, linux uses user ID, (UID) or group id (GUID) instead of usernames. These are numeric values assigned to each user for all permission related tasks.
	1.4 Which user stands above all when it comes to Linux users and groups?
		The user with the most power is the root user or superuser.
